Git核心概念

1.仓库（Repository）

本地仓库：项目根目录下的.git文件夹，存储所有版本信息。
远程仓库：托管在GitHub/Gitee等平台的云端仓库，用于备份和共享。

2. 提交（Commit）

一次提交代表一次完整的版本更新，包含修改说明、时间、作者等信息。
重要原则：提交前需经过测试，确保代码可运行。

3. 分支（Branch）

默认分支为master或main，可创建新分支开发新功能，避免污染主分支。
比喻：分支就像平行宇宙，互不干扰，最终可合并。

4. 暂存区（Staging Area）

提交前的“缓冲区”，选择需要纳入版本管理的文件。
用途：避免一次性提交所有改动，灵活控制版本内容。

Git基础操作

1. 初始化仓库

bash
git init # 将当前目录变为Git仓库

2.查看状态

bash
git status # 显示未跟踪/已修改的文件

3. 添加文件到暂存区

bash
git add file # 添加单个文件
git add . # 添加所有改动

4. 提交版本

bash
git commit -m "提交说明" # -m后跟简明的描述

5. 查看提交历史

bash
git log # 显示完整提交记录
git log --oneline # 简化版历史

分支管理

1. 创建与切换分支

bash
git branch dev # 创建dev分支
git checkout dev # 切换到dev分支
git checkout -b dev # 创建并切换（合并上述两步）

2. 合并分支

bash
git checkout main # 切换回主分支
git merge dev # 将dev分支合并到main

3. 解决冲突

当多个分支修改同一文件时需手动处理冲突，Git会用<<<<<<<标记冲突位置。

远程仓库操作

1. 关联远程仓库

bash
git remote add origin 远程仓库URL

2. 推送本地代码

bash
git push -u origin main # 首次推送需加 -u
git push # 后续推送简化

3. 拉取远程更新

bash
git pull # 相当于 git fetch + git merge

常见问题与小技巧

1. 撤销操作

撤销未提交的修改：

git checkout -- file

撤回暂存区文件：

git reset HEAD

修改最后一次提交:

git commit --amend

2. 忽略文件

创建.gitignore文件，列出需忽略的文件/文件夹（如node_modules/, .DS_Store）。

克隆现有项目

bash
git clone <仓库URL> # 下载远程仓库到本地
